## What is TSS?

TSS (Tea Simple Smart CSS) is a runtime utility-first CSS engine. Unlike Tailwind CSS, which requires a build step to generate a stylesheet, TSS runs entirely in the browser — scanning your HTML, generating only the CSS rules you actually use, and injecting them into the page automatically.

```html
<!-- Drop in one script tag — no build step, no config file -->
<script src="https://cdn.jsdelivr.net/npm/tea-simple-smart-css/dist/tss.min.js"></script>

<!-- Now use utility classes directly in your HTML -->
<div class="flex flex-col gap-4 p-8 bg-surface rounded-lg shadow-md">
  <h1 class="text-3xl font-bold text-heading">Hello, TSS!</h1>
  <p class="text-base text-body">Styled automatically, with light/dark mode built in.</p>
</div>
```

## Key Features

  <Card title="Zero Config" icon="circle-check">
    No config files, no PostCSS plugins, no build pipeline. Just a single script tag.

  <Card title="Auto Light/Dark Mode" icon="moon">
    Semantic color tokens automatically switch between light and dark themes using `prefers-color-scheme`.
  </Card>
  <Card title="Runtime Generation" icon="bolt">
    CSS is generated on-demand — only rules for classes actually used in the DOM are created.
  </Card>
  <Card title="Dynamic Content" icon="rotate">

  </Card>
  <Card title="Tailwind-Compatible" icon="layer-group">

  </Card>
  <Card title="Semantic Tokens" icon="palette">
    Named semantic colors like `bg-surface`, `text-heading`, `border-outline` adapt automatically to the active theme.
  </Card>
</CardGroup>

## How it works

<Steps>
  <Step title="Script loads">
    TSS auto-initializes on `DOMContentLoaded`, injecting two `<style>` tags: one for CSS custom properties (the theme) and one for utility rules.
  </Step>
  <Step title="DOM scan">
    TSS scans every element's class list and parses each utility class name into a CSS declaration block.
  </Step>
  <Step title="CSS injection">
    Generated rules are batched via `requestAnimationFrame` and flushed into the utilities `<style>` tag for optimal performance.
  </Step>
  <Step title="Live observation">
    A `MutationObserver` watches for new elements and class attribute changes, so dynamic content is styled automatically.
  </Step>
</Steps>

## Light/Dark mode at a glance

TSS generates CSS custom properties for all semantic color tokens and exposes them at `:root`. Dark mode is driven by `prefers-color-scheme` automatically, and you can override it programmatically:

```javascript
// Toggle dark mode manually and persist to localStorage
TSS.toggleDark();

// Check the current version
console.log(TSS.version);
```

<Note>
  Theme preference is persisted to `localStorage` under the key `tss-theme` and restored on every page load.
</Note>

## Package builds

| Build | File | Use case |
|-------|------|----------|
| IIFE (unminified) | `dist/tss.js` | Development, debugging |
| IIFE (minified) | `dist/tss.min.js` | CDN / `<script>` tag in production |
| ES Module | `dist/tss.esm.js` | Bundlers (Vite, Webpack, Rollup) |
| TypeScript | `dist/index.d.ts` | Type-safe imports |
