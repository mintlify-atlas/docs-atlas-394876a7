## Palette colors

The `PALETTE` export provides the full color scale. Tokens follow the naming pattern `{color}-{shade}`. The 19 available color names are: `slate`, `gray`, `zinc`, `red`, `orange`, `amber`, `yellow`, `lime`, `green`, `emerald`, `teal`, `cyan`, `sky`, `blue`, `indigo`, `violet`, `purple`, `pink`, `rose`. Each color exposes shades: `50`, `100`, `200`, `300`, `400`, `500`, `600`, `700`, `800`, `900`, `950`.

Example rows from `blue` and `gray`:

| Token | Value |
|-------|-------|
| `blue-50` | #eff6ff |
| `blue-100` | #dbeafe |
| `blue-400` | #60a5fa |
| `blue-500` | #3b82f6 |
| `blue-600` | #2563eb |
| `blue-900` | #1e3a8a |
| `gray-50` | #f9fafb |
| `gray-300` | #d1d5db |
| `gray-500` | #6b7280 |
| `gray-700` | #374151 |
| `gray-900` | #111827 |

## Special values

`PALETTE` exports `white` (`#ffffff`), `black` (`#000000`), and `transparent` (`transparent`) as named entries. Additionally, `resolveColor()` in `parser.js` handles two keyword values that are not in `PALETTE` but are resolved directly:

| Token | Value | Example |
|-------|-------|---------|
| `white` | `#ffffff` | `bg-white`, `text-white` |
| `black` | `#000000` | `bg-black`, `text-black` |
| `transparent` | `transparent` | `bg-transparent`, `text-transparent` |
| `inherit` | `inherit` | `text-inherit` (resolved in parser, not in PALETTE) |
| `current` | `currentColor` | `text-current` (resolved in parser, not in PALETTE) |
