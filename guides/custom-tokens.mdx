title: "Custom tokens"
description: "Override TSS semantic color tokens to match your brand."
---

TSS semantic color tokens are injected as CSS custom properties at `:root` when the script initializes. You can override any token with a `<style>` block placed after the TSS script tag.

## Override with CSS

Add a `<style>` tag **after** the TSS `<script>` tag to override tokens:

```html
<script src="https://cdn.jsdelivr.net/npm/tea-simple-smart-css/dist/tss.min.js"></script>

<style>
  /* Light mode overrides */
  :root {
    --tss-accent:  #7c3aed;   /* purple instead of default indigo */
    --tss-primary: #fafaf9;   /* warm white */
    --tss-base:    #f5f5f4;   /* warm gray page background */
  }

  /* Dark mode — media query */
  @media (prefers-color-scheme: dark) {
    :root {
      --tss-accent:  #a78bfa;
      --tss-primary: #1c1917;
      --tss-base:    #0c0a09;
    }
  }

  /* Dark mode — manual .dark class */
  html.dark {
    --tss-accent:  #a78bfa;
    --tss-primary: #1c1917;
    --tss-base:    #0c0a09;
  }

  html.light {
    --tss-accent:  #7c3aed;
    --tss-primary: #fafaf9;
    --tss-base:    #f5f5f4;
  }
</style>
```

<Warning>
Your `<style>` block must appear **after** the TSS `<script>` tag in the HTML. If placed before, TSS will overwrite your custom values when it initializes.
</Warning>

## Complete brand customization example

```html brand-theme.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Branded App</title>
  <script src="https://cdn.jsdelivr.net/npm/tea-simple-smart-css/dist/tss.min.js"></script>
  <style>
    :root {
      --tss-accent:   #7c3aed;
      --tss-link:     #7c3aed;
      --tss-ring:     #8b5cf6;
      --tss-base:     #fafaf9;
      --tss-surface:  #ffffff;
      --tss-primary:  #ffffff;
    }
    html.dark {
      --tss-accent:   #a78bfa;
      --tss-link:     #a78bfa;
      --tss-ring:     #8b5cf6;
      --tss-base:     #0c0a09;
      --tss-surface:  #1c1917;
      --tss-primary:  #1c1917;
    }
    @media (prefers-color-scheme: dark) {
      :root {
        --tss-accent:   #a78bfa;
        --tss-link:     #a78bfa;
        --tss-ring:     #8b5cf6;
        --tss-base:     #0c0a09;
        --tss-surface:  #1c1917;
        --tss-primary:  #1c1917;
      }
    }
  </style>
</head>
<body class="bg-base text-body">
  <h1 class="text-heading font-bold text-4xl">Branded with TSS</h1>
  <a href="#" class="text-link underline">A purple link</a>
  <button class="bg-accent text-primary px-4 py-2 rounded-lg" onclick="TSS.toggleDark()">
    Toggle Theme
  </button>
</body>
</html>
```

## Understanding the generated theme CSS

TSS injects a `<style id="tss-theme">` element on initialization containing all the CSS custom property definitions. You can inspect it in browser DevTools to see the exact generated output:

```javascript
// Inspect the generated theme CSS in the browser
const themeStyle = document.getElementById('tss-theme');
console.log(themeStyle.textContent);
// Outputs the full :root, @media, html.dark, html.light CSS blocks
```
