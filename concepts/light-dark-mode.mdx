---
title: Light & Dark mode
description: How TSS automatically generates adaptive CSS custom properties and lets users toggle or persist a theme preference.
---

TSS ships with a complete light/dark mode system out of the box. It requires zero configuration — semantic color tokens adapt automatically to the user's system preference, with optional manual override.

## How it works

During `init()`, TSS injects a `<style id="tss-theme">` element generated by `buildThemeCSS()` in `theme.js`. This stylesheet defines all semantic color tokens as CSS custom properties and sets up three layers of theme control:

```css
/* Light mode values — always present as the baseline */
:root {
  --tss-primary: #ffffff;
  --tss-secondary: #f3f4f6;
  --tss-surface: #ffffff;
  --tss-base: #f9fafb;
  --tss-heading: #111827;
  --tss-body: #374151;
  /* ... all other semantic tokens */
}

/* Automatically overrides when the OS is set to dark mode */
@media (prefers-color-scheme: dark) {
  :root {
    --tss-primary: #09090b;
    --tss-secondary: #18181b;
    --tss-surface: #18181b;
    --tss-base: #09090b;
    --tss-heading: #f9fafb;
    --tss-body: #d1d5db;
    /* ... all other semantic tokens */
  }
}

/* Manual override — takes precedence over media query */
html.dark {
  --tss-primary: #09090b;
  /* ... dark values */
}

html.light {
  --tss-primary: #ffffff;
  /* ... light values */
}
```

The `body` element also receives smooth transitions so theme switches don't flash:

```css
body {
  background-color: var(--tss-base);
  color: var(--tss-body);
  transition: background-color 0.2s ease, color 0.2s ease;
}
```

## Theme persistence

TSS reads and writes a `'tss-theme'` key in `localStorage` to remember the user's preference across page loads.

**On load**, `restoreThemePreference()` runs before `init()` and applies any saved preference before the first paint:

```js
function restoreThemePreference() {
  try {
    const saved = localStorage.getItem('tss-theme');
    if (saved === 'dark') {
      document.documentElement.classList.add('dark');
    } else if (saved === 'light') {
      document.documentElement.classList.add('light');
    }
  } catch (e) { }
}
```

**To toggle**, call `toggleDark()` from the public TSS API. It flips the class on `<html>` and saves the new preference:

```js
function toggleDark() {
  const html = document.documentElement;
  const isDark = html.classList.contains('dark');

  if (isDark) {
    html.classList.remove('dark');
    html.classList.add('light');
  } else {
    html.classList.remove('light');
    html.classList.add('dark');
  }

  try {
    localStorage.setItem('tss-theme', isDark ? 'light' : 'dark');
  } catch (e) { }

  return !isDark;
}
```

`toggleDark()` returns `true` if the page is now in dark mode, `false` if it switched to light. The `try/catch` blocks ensure the function degrades gracefully in environments where `localStorage` is unavailable (private browsing, sandboxed iframes).

To wire it up to a button:

```html
<button onclick="TSS.toggleDark()">Toggle theme</button>
```

## Priority order

When multiple signals exist simultaneously, TSS resolves the active theme in this order:

1. **`html.dark` / `html.light` class** — explicit class on the `<html>` element, set by `toggleDark()` or `restoreThemePreference()`. Highest priority; overrides the media query.
2. **`prefers-color-scheme` media query** — reflects the operating system or browser dark mode setting. Active when no manual class override is present.
3. **Light mode values on `:root`** — the unconditional baseline. Lowest priority fallback, always present.

## Semantic color tokens

<Tip>
  Always prefer semantic color tokens over raw palette colors for any page that needs dark mode support. Raw palette values (e.g., `bg-gray-100`) are static hex values and will not change with the theme.
</Tip>
