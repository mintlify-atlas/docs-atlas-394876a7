## What is TSS?

TSS (Tea Simple Smart CSS) is a utility-first CSS engine that runs entirely in the browser. Instead of generating a stylesheet at build time, TSS parses your HTML class names at runtime and injects the corresponding CSS rules directly into the page.

You write classes like `flex gap-4 text-heading bg-base` and TSS produces the CSS for exactly those classes — nothing more, nothing less.

## How it differs from Tailwind

| | TSS | Tailwind CSS |
|---|---|---|
| **Runs at** | Browser runtime | Build time |
| **Requires PostCSS** | No | Yes |
| **Requires purge/scan config** | No | Yes |
| **Works from CDN** | Yes | No |
| **Handles dynamic classes** | Yes (via MutationObserver) | No |
| **Bundle size impact** | None | Requires tooling |

Tailwind generates an optimized stylesheet during a build step by scanning your source files. TSS skips that entirely — drop in a script tag and your classes work immediately, including classes added dynamically by JavaScript.

## Who it's for

TSS is a good fit for:

- **Rapid prototyping** — add one script tag and start building without configuring a toolchain
- **CDN-hosted pages** — static HTML files, CodePen embeds, demos, and documentation examples
- **Lightweight pages** — marketing pages, landing pages, or internal tools where a full build pipeline is overkill
- **Dynamic content** — pages where class names are added at runtime by JavaScript (carousels, modals, server-rendered partials)

## Distribution formats

TSS ships three build outputs from the `dist/` directory:

- **`tss.js`** — unminified IIFE for development, exposes a global `TSS` object
- **`tss.min.js`** — minified IIFE for CDN production use
- **`tss.esm.js`** — ES module for bundlers such as Vite, Rollup, or esbuild
