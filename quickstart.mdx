description: "Build your first page with TSS in under 5 minutes."
---

TSS requires no build tools, no config files, and no CLI. Add one script tag and start writing utility classes.

<Steps>
  <Step title="Add the TSS script">
    Place the script tag inside `<head>` — before your page content — to avoid a flash of unstyled elements:

    ```html
    <script src="https://cdn.jsdelivr.net/npm/tea-simple-smart-css/dist/tss.min.js"></script>
    ```

    That's it. TSS auto-initializes on `DOMContentLoaded` with no further configuration.
  </Step>

  <Step title="Write your HTML with utility classes">
    Use TSS utility classes directly in your markup. Here's a complete working page:

    ```html index.html
    <!DOCTYPE html>
    <html lang="en">
    <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>My TSS Page</title>
      <script src="https://cdn.jsdelivr.net/npm/tea-simple-smart-css/dist/tss.min.js"></script>
    </head>
    <body class="min-h-screen bg-base p-8">

      <div class="max-w-2xl mx-auto">
        <h1 class="text-4xl font-bold text-heading mb-4">Welcome to TSS</h1>
        <p class="text-base text-body mb-6">
          A utility-first CSS engine that works without a build step.
        </p>

        <div class="bg-surface border border-outline rounded-lg p-6 shadow-md mb-6">
          <h2 class="text-xl font-semibold text-heading mb-2">Dark mode ready</h2>
          <p class="text-body text-sm">
            Semantic color classes like <code>bg-surface</code>, <code>text-heading</code>,
            and <code>text-body</code> automatically adapt to light and dark mode.
          </p>
        </div>

        <div class="flex gap-4">
          <button
            class="bg-accent text-primary px-6 py-2 rounded-lg font-semibold cursor-pointer transition-colors"
            onclick="TSS.toggleDark()"
          >
            Toggle Dark Mode
          </button>
          <button
            class="bg-secondary text-body px-6 py-2 rounded-lg font-semibold border border-outline cursor-pointer"
          >
            Secondary
          </button>
        </div>
      </div>

    </body>
    </html>
    ```

    Open this file in a browser — you'll see a styled page with working dark mode toggle, no build step needed.
  </Step>

  <Step title="Understand what happened">
    When TSS loaded, it:

    1. Injected a `<style id="tss-theme">` with CSS custom properties for all [semantic color tokens](/concepts/semantic-colors)
    2. Scanned every element's class list and generated CSS rules for each recognized utility class
    3. Flushed the generated CSS into a `<style id="tss-utilities">` tag
    4. Started a `MutationObserver` to handle any future DOM changes

    Classes like `bg-base`, `text-heading`, and `text-body` resolve to CSS custom properties (`var(--tss-base)`, `var(--tss-heading)`, `var(--tss-body)`) that automatically change value between light and dark mode.

    <Note>
      Unrecognized class names are silently ignored — TSS won't break if you mix in custom class names.
    </Note>
  </Step>

  <Step title="Toggle dark mode">
    Click the "Toggle Dark Mode" button you added in step 2. This calls `TSS.toggleDark()`, which:

    - Adds or removes the `dark` class on `<html>`
    - Saves the preference to `localStorage` so it persists across page reloads

    You can also call it from JavaScript:

    ```javascript
    // Returns true when dark mode is now active
    const isDark = TSS.toggleDark();
    console.log('Dark mode:', isDark);
    ```
  </Step>
</Steps>
